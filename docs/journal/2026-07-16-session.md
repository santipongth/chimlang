## Codex — Phase 8 M5 Core Engine/Retrieval และปิดเฟส

- บริบท/มติผู้ใช้: ผู้ใช้สั่ง “continue งานต่อ ทำที่เหลือ” หลัง M4 commit แล้ว จึงทำเฉพาะ
  P8-M5 ทั้งห้ารายการตาม brief และไม่เปิด Phase 9; รักษา `.gitignore` modified กับ `diagrams/`
  untracked ซึ่งเป็นงานเดิมของผู้ใช้ไว้ไม่แตะ
- อ่าน governance/architecture ใหม่: `AGENTS.md`, `CLAUDE.md`, STATE, PHASE8 brief,
  TECH-DECISIONS และ ADR-0001/0007/0008/0010/0011; ยึด LLM adapter เดียว, BudgetGuard,
  migration-only DDL, provenance/reproducibility, Thai-first และ no-raw-PII telemetry
- Typed Debate:
  - เพิ่ม move contract `claim`, `evidence`, `counterclaim`, `concession`, `question`
    พร้อม deterministic `move_id`, `parent_move_id`, `evidence_refs`; prompt/JSON Schema และ DB snapshot
    รองรับ lineage โดย snapshot เก่าไม่ถูกแก้และอ่านเป็น `legacy_unverifiable`
  - เพิ่ม deterministic verifier ตรวจ unique/schema, parent ต้องมาก่อน, evidence ID, evidence move ไม่มี
    citation, unsupported numeric claim, concession/contradiction; เก็บ violations ดิบและ lineage graph
  - analyst synthesis call เดิมทำหน้าที่ judge ด้วย เพื่อลด call/cost; judge ครบ citation,
    contradiction, unsupported claim และ schema assessment และมี deterministic severity floor กัน analyst
    ลดระดับ verifier finding
- Reflection:
  - opt-in เท่านั้น, เก็บใน run เดียว ไม่เขียน long-term memory; policy จำกัด 2 calls, input 2,400 chars,
    output 220 tokens และทุก call ใช้ analyst tier ผ่าน adapter/BudgetGuard
  - เพิ่ม offline paired smoke harness เพื่อพิสูจน์ bounds/measurement path; รายงานชัดว่า fixture ที่กำหนด
    ล่วงหน้าไม่พิสูจน์ causal efficacy ของ reflection
- Retrieval:
  - เพิ่ม embedding model/dimension ใน Settings/env และ `LLMAdapter.embed()` ที่คิดราคา fail-closed,
    เช็ก marginal cost ก่อน provider call, บันทึก model version/dimension และรวม actual cost ใน run/monthly spend
  - เปลี่ยน monthly spend ledger ของ debate เป็นบันทึกต่อ provider call หลังได้ usage จริง เพื่อให้งานที่
    ล้มเหลวหรือ call ที่ชน runtime cap ยังถูกนับต้นทุน; ตัด aggregate post-run write เดิมเพื่อไม่ให้นับซ้ำ
  - เพิ่ม migration pgvector table, HNSW expression index สำหรับ 1536d; dimension อื่นทำ exact pgvector
    พร้อม provenance ไม่อ้างว่า HNSW
  - แก้ BM25 ให้ tokenize Thai character trigrams เมื่อไม่มีเว้นวรรค; hybrid ใช้ Reciprocal Rank Fusion
    ระหว่าง BM25/vector rank; embedding ไม่ตั้ง/ล้ม = explicit `bm25_fallback_embedding_unavailable`
  - vector retrieval ตรวจ coverage ก่อน query; ถ้า index สำเร็จเพียงบาง chunk จะไม่ใช้ผล vector บางส่วน
    แต่ fallback BM25 พร้อม `embedding_coverage_incomplete` เพื่อไม่ให้ ranking ดูสมบูรณ์เกินจริง
- Benchmark:
  - เพิ่ม fixture ไทยและ runner ครอบ retrieval, evidence relevance, subgroup fidelity,
    social-desirability, future-event calibration และ reflection bounds
  - ผลดิบ: retrieval n=2 Recall@5/MRR=1.0; evidence n=2 unsupported=1; subgroup n=3
    MAE=.013333; social n=2 direction accuracy=1.0; future calibration n=4 Brier=.0825,
    baseline=.25, skill=.67, **ECE=.275**; เก็บข้อจำกัด/sample size ตรง ๆ ใน
    `docs/reports/P8-M5-benchmarks.md` ไม่มี threshold ย้อนหลัง
- Observability:

  - เพิ่ม Prometheus counters/histograms ที่ `/metrics`, provider call metadata table และ
    `/observability.json`; metadata มี provider/operation/tier/status/latency/tokens/cost/error kind/model

  - Insights มี queue/provider health, success rate, latency, cost และ failure taxonomy; Settings ตั้ง
