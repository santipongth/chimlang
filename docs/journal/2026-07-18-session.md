# Session 18 ก.ค. 2026 — Decommission 4 workspaces

## สิ่งที่ผู้ใช้สั่ง

ผู้ใช้สั่งให้ลบเมนู “โปรเจกต์และหลักฐาน”, “ห้องทดลองความน่าเชื่อถือ”, “ซ้อมแถลงข่าว” และ
“ทดสอบกับผู้ใช้” พร้อม clean ไฟล์ โค้ด และฐานข้อมูลที่เกี่ยวข้อง เพื่อให้ระบบพร้อมใช้งานจริง

## คำอธิบายและขอบเขตที่ใช้

- งานนี้เป็นการถอด capability จริง ไม่ใช่ซ่อน sidebar จึงไล่ dependency ตั้งแต่ React route ถึง PostgreSQL
- แจ้งผู้ใช้ก่อนลงมือว่า Prediction/Finding และ audit log เป็น append-only ตาม governance จึงไม่ลบประวัติ
- ตีความ “เกี่ยวข้อง” เป็น operational workspace ของ Project/Evidence, Validation dataset/report และ
  Rehearsal session รวม Usability UI/runner ไม่รวม evidence lineage/direct governed sources ของ run,
  Calibration, per-run validation, immutable run manifests หรือ financial ledger
- working tree มี user-owned changes เดิมใน `.gitignore`, `simulation/debate.py`, `tests/test_phase6.py`,
  `web/src/i18n.tsx`, `web/src/pages/RunDetail.tsx`, `web/tsconfig.tsbuildinfo`, `diagrams/` และ
  `ExecutiveReadout.test.tsx`; แก้ `i18n.tsx` เฉพาะ 4 nav keys และรักษาการแก้อื่นทั้งหมด

## งานที่ทำ

1. เขียน ADR-0019 เป็นมติ Accepted และปรับสถานะ ADR-0015/ADR-0016 ส่วนที่ถูก supersede
2. ลบ React pages/routes/menu/API client ของทั้ง 4 surface และลบ proxy/E2E stubs ที่ไม่ใช้แล้ว
3. ลบ FastAPI routers, project/validation/rehearsal stores, rehearsal engine/CLI, validation runners และ tests
4. ตัด `project_id`/`evidence_set_id` ออกจาก RunBody, manifest normalization และ generated TypeScript contract
5. ถอด project linkage จาก PopulationSet ใหม่ โดยคง immutable PopulationSet contract
6. ปรับ `scripts/live_workflow.py` เป็น PopulationSet → Run → Result → Export; Debate ใช้ direct governed source
7. เพิ่ม migration `2026-07-18-remove-project-validation-rehearsal-usability-v1`; historical migrations ที่เคย
   import โมดูลถูกเปลี่ยนเป็น no-op marker เพื่อให้ fresh install ยัง migrate ได้
8. เพิ่ม regression tests ว่า OpenAPI/RunBody ไม่มี contract เดิม, 14 ตารางหาย และ deep links เดิมเข้า 404

## การล้างฐานข้อมูล

ก่อน migration ตรวจ `pg_stat_user_tables` แล้วทั้ง 14 ตารางเป้าหมายมี 0 rows จึงไม่มีข้อมูลผู้ใช้ถูกลบจริง
ในรอบนี้ แต่ schema ถูกลบถาวรตามคำสั่ง ได้แก่ project 7 ตาราง, validation 4 ตาราง และ rehearsal 3 ตาราง
พร้อมลบ `population_sets.project_id`/index. หลัง migration `removed_remaining=0` และตรวจว่า immutable ledgers
หลัก 5 กลุ่มยังมีอยู่ครบ. รัน migration ซ้ำได้ผล no-op.
