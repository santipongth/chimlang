# Session 18 ก.ค. 2026 — Executive Readout ไม่แสดงผล

## สิ่งที่ผู้ใช้สั่ง

ผู้ใช้ขอให้ตรวจสอบ แก้ไข และอธิบายสาเหตุที่ Executive Readout ไม่แสดงผล

## หลักฐานและ root cause

- ตรวจ run จริง `debate-20260717-172412-853248-ea918868` และ
  `debate-20260717-171555-105793-c2ff832a`: ทั้งคู่มี status `complete`, manifest complete และ
  parser_mode `json_schema` แต่ `payload.synthesis` มีเพียง `bucket`, `pct`, confidence และ judge fallback
  ไม่มี `summary`, `distribution`, `key_drivers`, `risks`
- model/provider ส่งสมาชิกหนึ่งตัวของ distribution เป็น root object แม้ request ใช้ structured output
- backend เดิม `_load_json_object()` ยืนยันเพียงว่า decode แล้วเป็น dict จากนั้นเติม confidence/judge และถือว่า
  synthesis สำเร็จ จึงเป็น validation gap ระหว่าง provider capability กับ business contract
- frontend เดิมอ่าน `p.synthesis.summary` โดยตรง จึง render heading/เนื้อหาว่าง; ปัญหาไม่ใช่ CSS หรือ route

## งานที่แก้

1. นิยาม Pydantic contract แบบ `extra=forbid`, non-empty text, confidence/pct bounds และ list ขั้นต่ำ
2. validate analyst response ในเครื่องทุกครั้ง แม้ provider บอกว่าใช้ `json_schema`
3. ถ้า response แรกผิด contract ให้ retry ได้หนึ่งครั้งแบบ parser path; adapter estimate สำรอง call นี้และทุก call
   ยังผ่าน BudgetGuard/monthly reservation ตามเดิม พร้อม provenance `analyst_attempts`/`parser_mode`
4. ถ้ายังผิด contract ให้ synthesis เป็น `analyst_failed` และ API fail run โดยไม่สร้าง mechanical fallback
5. frontend guard ตรวจ summary/confidence/distribution/drivers/risks ก่อน render; legacy run ที่ไม่ครบแสดงคำเตือน
   และปุ่มรันใหม่ด้วย frozen input แทน readout ว่าง
6. เพิ่ม unit/integration/browser regressions ครอบ fragment `{bucket,pct}`, retry success, retry fail และ UI guard

## ผล live และการตรวจสอบ

- deploy Compose image ใหม่แล้วทุก service healthy
- live Debate 2 agents × 1 round ผ่าน BudgetGuard: estimate USD 0.000775, actual USD 0.000539
- run `debate-20260717-235347-756596-7bcc81cd` complete และเก็บ summary จริง, confidence 0.85,
  distribution 1, drivers 5, risks 4, judge warn, analyst_attempts 1, parser_mode json_schema, manifest complete
- `uv run pytest -q`: 411 tests ผ่าน
- `uv run ruff check .` + `uv run ruff format --check .`: ผ่าน

## สถานะส่งต่อ

run เก่าที่ payload ไม่ครบไม่ถูกแก้ย้อนหลังเพราะ manifest/snapshot เป็น immutable; UI จะระบุว่าใช้ไม่ได้และให้
