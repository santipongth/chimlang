ย้ายมา UI ได้หมด | ผู้ใช้เลือกผ่าน AskUserQuestion: **เข้ารหัสก่อนเก็บ** + **งบต่อรัน+รวมเดือน**

มาสก์เท่านั้น), ราคาโมเดลแก้/เพิ่มจาก UI, งบเดือน `core/llm/budget.py`, tests 302→309

## ช่วงที่ 6 — Phase 7: News Desk + Media Diet (ไอเดียผู้ใช้ → วิจัย → ทำจริง)

**ผู้ใช้เสนอ**: ให้ AI agent แต่ละตัว tool calling ดึงข้อมูลล่าสุดจากเน็ตเอง — "น่าจะเป็น
แนวทางใหม่ที่ algorithm ต่างประเทศไม่มี"

**ผลวิจัย (ตรวจความใหม่อย่างซื่อสัตย์)**:
- agentic search เดี่ยว: มีแพร่หลาย (MA-RAG, TURA) | debate+retrieval: มีในงานวิจัย (Madam-RAG)
- **social simulation ที่ agent ดึงเน็ตสดระหว่างรัน: ยังไม่มีใครทำ** — OASIS/GenSim/AgentSociety
  ใช้วิธีผู้วิจัย inject เข้า environment
- จุด novel จริง = **persona-conditioned retrieval (media diet)**: แต่ละกลุ่มเห็นเน็ตไม่เหมือนกัน
  ตามพฤติกรรมสื่อของกลุ่ม (selective exposure) — เข้าทาง Thai Social Fabric ที่มี channel_mix อยู่แล้ว

**ความเสี่ยงที่ชี้ + ทางแก้**: reproducibility พัง (→ snapshot-first), hindcast รั่ว (→ gate เดิม),
PII (→ gate ทุกชิ้น), ต้นทุนระเบิดถ้า per-agent ค้นเอง (→ โต๊ะข่าวกลาง + intent dedupe),
bias ของ search engine (→ provenance)

**มติผู้ใช้**: approve "โต๊ะข่าวกลาง + media diet", RSS + Search API ตั้งแต่ M1, ทำครบ M1-M4

**ที่ทำ** (ADR-0008, PHASE7-BRIEF): `simulation/newsdesk.py` — `gather()` จุดเดียวแตะเน็ต
(hindcast gate ก่อน I/O, PII ทุกชิ้น, snapshot `news_items`, caps 30/8/3), `segment_feed()`
media diet ตาม channel_mix (deterministic), debate มี `want_to_know` → โต๊ะข่าวค้นระหว่างรอบ,
`live_news` ใน POST /runs, UI toggle 🌐 + ข่าวใน tab หลักฐาน | tests 312→320
(รวม test: mock httpx ระเบิดถ้า replay แตะเน็ต)
