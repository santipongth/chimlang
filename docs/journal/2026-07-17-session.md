### สิ่งที่ผู้ใช้สั่งและมติที่ถือเป็นฐาน

ผู้ใช้ส่ง roadmap Phase 9 ที่วิเคราะห์จาก codebase/research แล้วและสั่งให้ implement ใน fresh context.
Roadmap กำหนด 8–12 สัปดาห์ แบ่ง M1–M3 และระบุชัดว่า M1 เป็น trust gate: เมื่อรายงาน migration,

เหตุผลหลักที่อธิบายระหว่างทำงาน: ความเสี่ยง pilot ปัจจุบันอยู่ที่ trust contract, cancellation race,
asynchronous workflow, SSRF และ navigation มากกว่าการเพิ่ม simulation engine. คำว่า replay ต้องไม่เกินจริง:
provider pin seed ได้ best-effort เท่านั้น ตัว freeze ที่ตรวจได้คือ input/result snapshot + manifest hash.

   request/config/artifact hashes, code/prompt/model/pricing/governance snapshots และ append-only store.

   posts ถูกปฏิเสธหลัง terminal และ worker เช็ก cancellation ก่อนทุกจุดเขียนสำคัญ.

## Codex — Phase 9 M2/M3 Projectized Trust Workflow

### มติผู้ใช้และขอบเขต

ผู้ใช้อนุมัติชัดเจนว่า “อนุมัติทำ เริ่ม M2/M3 ต่อ” หลัง M1 ผ่าน จึงเดินต่อเฉพาะ roadmap Phase 9
โดยไม่เพิ่ม engine, Concordia, public-GA หรือ 5,000-agent tier อธิบายตั้งแต่ต้นว่า MIRACL ต้องรันจาก
dataset จริงและ usability/human-panel outcome ต้องมาจากมนุษย์จริง ห้าม fabricate

### งาน implementation

1. เพิ่ม Project/Case workflow พร้อม append-only revisions และ stage gate; Project ผูก frozen evidence,
   runs, decision และ resolution
2. เพิ่ม Evidence Library: PDF/DOCX/TXT/CSV, URL/RSS, version/dedup/source health/PII counts และ
   immutable EvidenceSetV1; direct PII block ก่อน persist, external source ผ่าน SafeOutboundFetcher แล้ว
   redact-and-verify; run ตรวจ set hash และ materialize เข้า RunSpec ก่อน enqueue
3. เพิ่ม Validation Lab/Resolution Inbox: dataset/report registry, consent-based human-panel import,
   prediction owner events, Brier/ECE/reliability/CI และ multi-model aggregate cost preflight แบบ opt-in
4. เพิ่ม event-sourced Rehearsal API/UI: turn/question/answer/reactions, pause/resume CAS, operator decision,
   transcript และ scorecard; audit พบ adapter ยังไม่ bind session ledger จึงแก้ให้ reserve budget ทั้ง session
   แบบ transactional, record actual ทุก call และ release ส่วนเหลือเมื่อ finish; audit รอบท้ายพบ concurrent
   `/next` อาจเสียเงินซ้ำก่อน append lock จึงเพิ่ม tokenized PostgreSQL operation lease ครอบ provider calls
5. เพิ่ม HashRouter deep links `/projects/:id`, `/validation`, `/rehearsals/:id`; mobile nav, locale formatting,
   file upload และ Run Detail tabs Result/Evidence/Uncertainty/Validation/Audit; แยก shell API ทำให้ feature
   routes lazy และ initial index ลดจากระหว่างพัฒนา 98.02 เหลือ 85.81 kB
6. เพิ่ม migrations project/evidence, validation และ rehearsal; ADR-0015, named Pydantic responses,
   generated TypeScript client, unit/E2E regression
