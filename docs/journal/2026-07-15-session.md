## Codex — ลงมือแก้ทุกข้อที่แนะนำจาก review

ผู้ใช้สั่งต่อหลัง review: "คุณลงมือทำทุกอย่างที่แนะนำมาให้ฉันหมดเลย ทั้งหมด" จึงถือเป็นมติให้ implement hardening ที่แนะนำ แม้ Phase 7 จะปิดแล้ว โดยยังยึด governance เดิมและไม่แตะงานค้างของผู้ใช้ (`.gitignore`, `diagrams/`).

งาน backend ที่ทำ:
- เพิ่ม Pydantic lower bound `agents >= 1` ให้ `/runs`, `/jobs/whatif`, `/gallery/share`, `/watchlists` เพื่อ reject `agents <= 0` ก่อนเข้า simulation/DB และเพิ่ม `PersonaFactory.sample()` ให้ fail-fast เมื่อ `n <= 0`; ตอนแรกตั้ง `ge=10` แล้ว full pytest จับ regression เพราะ debate test เดิมใช้ 6 agents จึงลดเป็น `ge=1` ตามเป้าหมายจริงคือกันศูนย์/ติดลบ ไม่เปลี่ยน contract เดิมเกินจำเป็น

## Codex — ทำ backend performance + UI/UX professional pass ตามอนุมัติผู้ใช้

ผู้ใช้ถามต่อว่ามีข้อแนะนำอะไรอีกเพื่อเพิ่มประสิทธิภาพ/ปรับ UI/UX ให้ดูดีขึ้น มืออาชีพ และไม่เหมือนใคร จึงเสนอ backend/system 6 ข้อ และ UI/UX 7 ข้อ รวม feature เด่น 3 ข้อ จากนั้นผู้ใช้สั่ง: "ทำทั้งหมดเลย ทั้ง Backend / ระบบ และ UI/UX ให้ดูมืออาชีพขึ้น ทำทุกข้อ ฉันอนุมัติ" จึงลงมือ implement รอบใหญ่โดยคุมให้ไม่เพิ่ม dependency ใหม่และไม่พลิก ADR เดิม

Backend/system ที่ทำ:
- `core/runstore.py`: ขยาย lifecycle ของ `sim_runs` เป็น `queued/running/complete/error/canceled`, เพิ่ม `job_id`, `queued_at`, `started_at`, `finished_at`, `progress`, `progress_message`; `setup()` ทำ ALTER TABLE แบบ idempotent สำหรับ DB เดิม

- เพิ่ม `/runs/{run_id}/cancel` และ `/runs/{run_id}/retry`; cancel update DB + best-effort revoke Celery + audit; retry สร้าง run ใหม่จาก config เดิม (ไม่เก็บ raw sources ซ้ำเพื่อเลี่ยงข้อมูลอ่อนไหว)
- เพิ่ม `/run-metrics.json`: status counts, avg queue wait, avg runtime, errors 24h, source/news status counts, monthly spend
- `simulation/sources.py`: เพิ่ม `external_fetch_cache` สำหรับ URL/RSS content TTL 6 ชั่วโมง และ helper `setup_sources()`; retrieval เปลี่ยนเป็น hybrid deterministic แบบไม่เพิ่ม cost: 3-gram overlap + term overlap + label boost
