Findings ที่เตรียมรายงานผู้ใช้:
1. 2. `tests/test_phase6.py` ทำ PostgreSQL readiness check ตอน import ด้วย `psycopg.connect(DSN)` ไม่มี `connect_timeout`; เมื่อ dev stack ไม่เปิด `pytest`/collect ค้างก่อน skip ได้ เป็นสาเหตุที่ full suite timeout ใน session นี้
3. News Desk กลืน exception จาก Tavily/RSS ก่อน snapshot (`continue`) ทำให้ evidence/replay ไม่มี record ว่า provider ล้ม แหล่งข่าวพัง หรือไม่มี key; ไม่ใช่ leak/PII bug โดยตรง แต่ลด auditability ของ Phase 7 ที่ประกาศ snapshot-first/provenance

## Codex — ลงมือแก้ทุกข้อที่แนะนำจาก review

ผู้ใช้สั่งต่อหลัง review: "คุณลงมือทำทุกอย่างที่แนะนำมาให้ฉันหมดเลย ทั้งหมด" จึงถือเป็นมติให้ implement hardening ที่แนะนำ แม้ Phase 7 จะปิดแล้ว โดยยังยึด governance เดิมและไม่แตะงานค้างของผู้ใช้ (`.gitignore`, `diagrams/`).

งาน backend ที่ทำ:
- เพิ่ม Pydantic lower bound `agents >= 1` ให้ `/runs`, `/jobs/whatif`, `/gallery/share`, `/watchlists` เพื่อ reject `agents <= 0` ก่อนเข้า simulation/DB และเพิ่ม `PersonaFactory.sample()` ให้ fail-fast เมื่อ `n <= 0`; ตอนแรกตั้ง `ge=10` แล้ว full pytest จับ regression เพราะ debate test เดิมใช้ 6 agents จึงลดเป็น `ge=1` ตามเป้าหมายจริงคือกันศูนย์/ติดลบ ไม่เปลี่ยน contract เดิมเกินจำเป็น
- เพิ่ม `connect_timeout=2` ใน PostgreSQL readiness check ของ tests ที่ใช้ skip marker เพื่อไม่ให้ pytest collection ค้างเมื่อ dev stack ไม่เปิด
- เพิ่ม URL guard กลางใน `simulation.sources.validate_external_url()` บล็อก scheme ที่ไม่ใช่ http(s), localhost, `.localhost`, และ private/loopback/link-local/reserved/multicast IP literal ก่อน fetch source/RSS ลด SSRF/local access risk
- ปรับ News Desk ให้ snapshot provider failure ลง `news_items` เป็น `status=error` หรือ `status=skipped` แทนการกลืน exception ของ RSS/Tavily; replay/evidence จึงเห็นว่าข่าวล้ม ไม่มี key หรืออ่าน feed ไม่สำเร็จ
- เพิ่ม persistent run queue: `POST /runs/async` enqueue ผ่าน Celery task `chimlang.persistent_run`, task เรียก path เดียวกับ `POST /runs` ด้วย synthetic principal ที่รักษา election verification, และ `GET /run-jobs/{job_id}` สำหรับ poll status/result/error

งาน frontend ที่ทำ:
- Wizard New Run เปลี่ยนไปเรียก `/runs/async`, แสดงสถานะ queued/running, poll `/run-jobs/{job_id}` จนได้ `run_id` แล้วเปิด RunDetail; eager mode ใน test/dev ยังคืน result ทันที
- Settings ดึง `/health/deep` มาแสดง dependency health แยก postgres/redis/neo4j
- RunDetail evidence tab เพิ่ม count/status pills สำหรับ news (`ready/blocked/error/skipped`) และ sources (`ready/blocked/error/empty`) เพื่อให้ auditability เห็นตั้งแต่หน้า UI

Tests/verification ที่เพิ่ม:
- `tests/test_phase6.py`: reject agents 0/-1 ก่อน DB, URL guard block localhost/private IP
- `tests/test_newsdesk.py`: provider failure/no Tavily key ต้องถูก snapshot เป็น error/skipped evidence
- `tests/test_jobs.py`: eager `/runs/async` สร้าง persistent run สำเร็จ, `/run-jobs/{job_id}` report failure ได้
