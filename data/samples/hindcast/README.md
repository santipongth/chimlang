# hindcast — ชุดเหตุการณ์อดีตสำหรับ M1 (PoC Data Cutoff) และ benchmark TRUST-03

แต่ละเหตุการณ์ = 1 โฟลเดอร์ ประกอบด้วย `before/` (ข้อมูลก่อน cutoff), `outcome.md`, `meta.yaml` (cutoff_date, วิธีวัดผล)

## ชุดที่มีตอนนี้ (ร่างโดย Claude, 5 ก.ค. 2026 — รอ user review)

| โฟลเดอร์ | เหตุการณ์ | cutoff | เหตุที่เลือก |
|---|---|---|---|
| `2565-bkk-governor-election/` | เลือกตั้งผู้ว่าฯ กทม. 22 พ.ค. 2565 | 2022-05-20 | ผลชัดเจน วัดง่าย อยู่ใน training data ของ LLM แน่นอน → leak test มีความหมายจริง |

## ⚠️ ข้อจำกัดที่ต้องรู้ก่อนใช้

1. **ไฟล์ `before/` เป็นบทสรุปที่เรียบเรียงย้อนหลัง** จากข้อเท็จจริงสาธารณะก่อน cutoff — ไม่ใช่บทความข่าวจริง ณ เวลานั้น เพียงพอสำหรับ PoC leak test (เป้าหมายคือทดสอบว่า filter กันความรู้ใน training data ของ model ได้ไหม) แต่การทำ hindcast benchmark อย่างเป็นทางการควรแทนที่ด้วยเอกสารจริง
2. **ตัวเลขใน `outcome.md` เป็นค่าโดยประมาณ** — ตรวจทานกับแหล่งทางการ (กกต. / มติ กสทช.) ก่อนใช้เป็นเกณฑ์วัดผล
3. **ชุดเลือกตั้ง**: output ระดับ aggregate เท่านั้น ติดป้าย simulation_estimate / not_field_poll ใช้เป็น benchmark ภายใน ห้ามเผยแพร่เป็นการทำนายเลือกตั้ง (GOV-02 / กฎเหล็กข้อ 6)
4. `outcome.md` **ห้ามป้อนเข้า simulation ระหว่างรัน** — hindcast runner ต้องโหลดเฉพาะ `before/` และ block ทุกอย่างหลัง cutoff (fail-closed)
5. Exit criteria ของ Phase 0 ต้องการ ≥ 3/5 เหตุการณ์ — จะเพิ่มชุดที่ 3–5 ใน M4 หลัง M1 ผ่าน gate
